# 高并发网络锁控系统架构

## 项目概览

基于 Flask + PyQt 的企业级高并发网络锁控系统，在复杂的分布式环境中实现稳定的访问控制和实时状态同步。

## 核心技术栈

- **后端框架**: Flask + Gunicorn + Nginx
- **前端**: PyQt5 / PyQt6 跨平台 GUI
- **通信协议**: WebSocket + TCP 双通道
- **数据库**: PostgreSQL + Redis 缓存
- **部署**: Docker 容器化 + Kubernetes 编排

## 系统架构

### 整体设计

```
┌─────────────────────────────────────────┐
│          Client Layer (PyQt)            │
│  - 本地缓存  - UI 状态管理 - 离线模式   │
└──────────────┬──────────────────────────┘
               │ WebSocket / TCP
┌──────────────▼──────────────────────────┐
│     Gateway Layer (Nginx + LB)          │
│  - 负载均衡  - SSL/TLS - 流量控制      │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│    Application Layer (Flask Cluster)    │
│  - 请求路由  - 业务逻辑 - 状态机      │
└──────────────┬──────────────────────────┘
               │
        ┌──────┴──────┐
        │             │
    ┌───▼──┐    ┌────▼───┐
    │Redis │    │PostgreSQL│
    │Cache │    │Database │
    └──────┘    └─────────┘
```

## 关键特性

### 1. 高并发处理
- 单机支持 **10K+** 并发连接
- 基于事件循环的异步 I/O
- 连接池和消息队列优化

### 2. 分布式一致性
- 使用 Redis 实现分布式锁
- 基于 Raft 共识的状态同步
- 多副本数据容错

### 3. 实时状态同步
- WebSocket 长连接推送更新
- 消息去重和顺序保证
- 秒级锁状态反馈

### 4. 安全机制
- JWT Token 认证
- 加密传输（AES-256）
- 操作审计日志

## 部署方案

### 开发环境
```bash
# Flask 开发服务器
python app.py

# PyQt 客户端
python client.py
```

### 生产环境
```bash
# Docker 部署
docker-compose -f docker-compose.prod.yml up -d

# Kubernetes 部署
kubectl apply -f k8s/
```
